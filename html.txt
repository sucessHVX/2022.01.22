HTML : 웹에서 사용하는 문서를 만드는 언어.
	HTML로 만든 문서는 다양한 정보(텍스트, 이미지, 비디오, 오디오 등)를 담고 있으며,
	사용자는 이 문서에서 원하는 정보를 얻고, 링크를 통해 현재 문서에서 다른 문서로 이동 가능.
하이퍼텍스트 : 링크로 연결되어 있는
마크업 : 문법적 특성이 있는(태그의 형태로 되어 있는)
랭귀지 : 컴퓨터 프로그래밍 언어

HTML로 작성된 문서는 브라우저를 통해 내용이 나타남, 사용자는 브라우저를 통해 HTML 문서의 정보 확인.
즉 HTML은 사용자와 웹브라우저 간의 상호작용을 위해 필요한 언어

각 브라우저에는 엔진이 있고 이 엔진이 HTML을 해석하고 CSS에 정의된 스타일로 화면에 배치.
즉 이 엔진은 정보(HTML)와 서식 정보(CSS)등을 해석하여 사람이 눈으로 보고 읽을 수 있는 문서로 표시하는 웹 브라우저의 핵심기능 담당
웹 공부할때 접하게 되는 WEBKIT(웹킷) 기반(구글, 사파리) 같은 것은 엔진 이름

엔진은 브라우저 엔진과 렌더링 엔진으로 분류됨
우리가 만들 HTML문서를 해석하는 엔진이 렌더링 엔진

렌더링 엔진의 동작과정은 작성된 HTML을 트리 형태로 작성한 다음 CSS처리가 이루어지고
이 두가지가 합쳐져 랜더 트리가 생성된다.
그 다음 적용된 스타일로 랜더 트리가 해당 위치에 배치되고 마지막으로 화면에 사용자가 볼 수 있게 나타난다.

쉽게 말해서 브라우저가 HTML과 CSS/자바스크립트를 합쳐 크기를 정하고 배치시켜 화면에 나타나게 해주는 것
브라우저가 HTML과 CSS명세에 따라 파일을 해석해서 나타내주기 때문에 애플, 모질라, 오페라, 구글 등 주요 업체들이
HTML 표준에 따라 브라우저를 만들고 있다.

HTML로 만들어진 웹 페이지 속에는 수많은 정보가 들어있다.
이런 정보를 수집하는 기계를 웹 크롤러 라고 하고,
이 웹 크롤러가 웹을 탐색하며 정보를 수집하여 저장하는 일을 크롤링 이라고 한다.

웹 크롤러 : HTML 구조로 웹 페이지의 콘텐츠 내용을 확인한다.
즉 정보를 수집하기 위해 웹 페이지 속의 HTML을 확인하고 분석한다.
예를 들어 <section> 태그 안에 <h1> 태그가 작성되어 있으면 제목이 있는 콘텐츠임을 웹 크롤러가 인지하고
콘텐츠를 분류하여 저장한다.

웹 퍼블리셔가 웹 페이지 속 정보를 의미 있는 태그로 작성하면 웹 크롤러는 정보의 의미를 명확하게 인식할 수 있다.
그리고 웹 크롤러는 그 내용을 저장하고 있다가 요청된(검색된) 내용에 맞게 사용자에게 제공한다.

이것이 검색 엔진 최적화가 필요한 이유이다. 따라서 검색 엔진 최적화를 위해서는 단순히 <meta> 태그의
키워드나 타이틀을 작성하는 데 국한하지 않고 콘텐츠의 구조와 의미까지 구분하여 html을 작성해야 한다.
단순히 화면에 콘텐츠를 표시하여 보이게 하는 것만을 목표로 삼을 것이 아니라 웹 크롤러가
html의 구조와 내용을 명확하게 파악할 수 있도록 작성해야 한다.

html을 작성할 때는 태그가 가진 의미를 이해하고 작성해야 한다.
html을 작성한다는 것은 다른 의미로 생각하면 웹 브라우저에서 실행되는 프로그램을 구현하는 일이다.
단지 디자인된 이미지를 브라우저에 나타나게 하는 것이 전부가 아니라 내가 만든 html이 원하는 결과를
가져올 수 있도록 의미에 맞게 태그를 작성하여 구현되도록 해야 한다.

사용자가 원하는 정보를 검색하면 그 내용에 맞게 링크를 제공하여 해당 웹페이지를 볼 수 있다.
이렇게 제공된 웹 페이지는 html로 작성되어 있다. 검색 엔진은 사용자에게 웹 페이지를 제공하기 위해 인터넷 상에서
크롤링하여 콘텐츠를 수집한다.

html 태그를 잘 쓰면 고객 유입, 상품 구매로도 이어진다.
자신의 사이트가 웹 크롤러에게 검색되게 하려면 html의 콘텐츠가 내용에 맞는 태그로 작성되고 표현되어야 한다.
그래야만 자신의 사이트가 사용자에게 더 많이 노출될 수 있다.
즉 웹 퍼블리셔가 웹 페이지 속에서 올바른 콘텐츠를 작성하면 웹 크롤러가 내용을 수집하고,
이렇게 저장된 내용이 검색 서비스에 사용되어 자신의 사이트가 우선순위로 노출되면 사용자 수가 늘어나고
광고 효과도 노릴 수 있어 구매까지도 기대할 수 있다.
이는 html문서 만으로도사이트의 차별화 전략을 세울 수 있다는 뜻이다.

예를들어 쇼핑몰의 경우 올바르게 작성된 콘텐츠가 실린 웹페이지를 만들어 놓았다면 많은 사용자가 
검색을 통해 이 쇼핑몰에 유입될 수 있고, 이후 매출에도 긍정적인 영향을 줄 수 있다.
또한 광고 목적의 웹 사이트라면 많은 사용자가 검색을 통해 해당 웹 사이트를 보고 입소문을 내어 이후에는
검색 엔진을 통하지 않고 사용자가 직접 사이트로 접속하여 광고를 볼 수 있다.

그래서 최근 검색 엔진 최적화와 관련하여 html작성에 대한 중요성이 높아지고 있다.
자신이 만든 웹 사이트가 웹 크롤러에 잘 걸리게 하기 위해서 검색 엔진 최적화를 위한 도입 페이지를
별도로 만들기도 하고, 마크업할 때는 <meta> 태그를 활용하여 웹 사이트의 요약 저옵를 작성하고
불필요한 이미지를 줄이고 <h> 태그를 사용하여 제목을 표시한다.
또한 <strong> </strong> 태그에 중요 키워드를 작성하고 <img> 태그에 alt 값을 넣는 등 콘텐츠를
의미 있는 태그로 구조화하여 작업한다.
이처럼 많은 노력으로 작성된 html은 검색 사이트에 잘 노출되어 광고효과를 볼 수 있다.

<meta>태그는 브라우저에 보이는 내용은 아니다. 하지만 검색 엔진 최적화를 하는데 중요한 태그이며
하나의 웹 사이트 정보를 담는 중요한 태그이다.
웹사이트를 만들때 중요하게 생각해야 할 요소 중 하나가 검색 엔진 최적화이다.
사이트를 만드는 목적이 혼자만의 비밀로 일기장을 만드는 것이 아니라면
많은 사람들에게 정보를 보여줄 수 있어야 한다.

검색 엔진의 웹 크롤러를 통해 사이트의 정보가 저장되고 있다.
웹 크롤러가 링크를 따라서 html 문서에 방문할 때 웹 사이트는 <meta> 태그에 지정한 요약 내용을 알려준다
검색 엔진의 작동 원리를 알면 좋겠지만 이는 잘 공개되지도 않고 변경되기도 합니다.
하나의 예로 몇년 전만 해도 <meta>태그의 keyword 속성으로 크롤링되었기 때문에 수많은 웹 사이트들이
무분별하게 keyword 속성을 작성했다.
구글은 더이상 크롤링할때 keyword 속성을 수집하지 않는다고 한다.

이렇게 내용이 변경되는 경우도 있으므로 웹 사이트를 작성할 때 검색 엔진에 최적화하기 위해서는 공식 가이드에
맞춰 작성하는 것이 가장 좋은 방법이다
검색엔진이 사용자의 질문에 최선의 답을 제공하기 위해서는 정확한 정보를 수집해야 한다.
그래서 규칙을 잘 지켜 작성된 콘텐츠를 선호하는 거다.
많은 요소들이 검색 엔진 최적화에 사용되지만 <meta> 태그의 내용은 검색 엔진이 웹 사이트를 파악할 때 기본으로
사용하고 있으므로 <meta> 태그를 올바르게 작성하는 것은 아주 중요하다.

<meta> 태그는 기본적으로 빈 요소이므로 시작 태그만 존재한다.
아래 소스는 html5의 인코딩을 지정하는 방법
<meta charset="UTF-8">

아래 소스는 IE의 버전 차이로 인해 다르게 보이지 않게 호환성 보기 기능이 활성화 되도록
지정하는 내용이며, 이것은 가장 최신의 렌더링 방법으로 웹문서를 브라우저에 표기해야함.
<meta http-equiv="X-UA-Compatible" content="IE=edge">

다음 viewport는 화면에 보이는 영역으로 너비를 현재 기기의 width로 정하고, 이에 맞춰
초기화면배율을 1로 지정한다는 의미
<meta name="viewport" content="width=device-width, initial=scale=1.0, maximum-scale=1.0,
   minimum-scale=1.0, user-scalable=no, target-densitydpi=medium-dpi">

html을 사용하여 홈페이지를 만들때 html을 의미 있는 태그로 만들어야 하는데, 이것을 시맨틱 웹이라고 한다.
html은 하나의 문서로 사용자, 브라우저, 검색 봇 모두에게 올바른 구조로 의미가 잘 전달되어야 한다.
html을 작성할 때 단순히 사전적인 의미로 태그를 사용해 html를 작성하는 것이 아니라
내용을 효율적으로 전달하는데 목적이 있음을 염두에 두고 작성하여야 한다.

html을 작성할때 고려해야 하는점
1. 일관성 있고 표준하된 코드를 사용해야함
약속된 언어로 공통구조를 마크업하여 표준화된 방법으로 제공한다면 일관성 있는 코드 사용으로 가독성을
높이고 구조는 그룹화되어 체계적으로 제공할 수 있다.
웹 표준의 기본은 html과 css/자바스크립트의 분리에 있다.
html로 내용을 만들고 css로 외형을 꾸미고 자바스크립트로 동작을 구분하여 만드는 것이다.
따라서 불필요한 태그를 줄여 의미있게 만들어야 한다.

2. 접근성을 지키며 작성해야함
웹에서 제공하는 모든 문서의 정보는 비장애인과 고령자도 동등하게 접근하고 이용할 수 있도록 만들어야한다.
이것을 웹 접근성이라 하며 웹 접근성 준수는 법적 의무 사항이다.
가령 시각장애인이 스크린 리더를 사용하여 웹에서 정보를 사용할때 내용의 흐름에 따라 이동할 수 있도록
페이지의 구조가 올바르게 되어 있어야 한다.
사용자가 손을 쓰기 어려워 마우스 동작이 불가능한 경우 키보드 탭으로 동작이 가능하도록 만들어야한다.
html의 접근성을 높이기 위해서는 css와 자바스크립트 없이도 html의 내용이 읽힐 수 있어야 한다.

3. 가독성을 고려하여 작성한다.
html로 작성된 코드가 사용자에게 전달되는 것에만 목적이 있는 건은 아니다.
여러명의 웹 퍼블리셔 사이에 전달되어 협업 과정을 통해 하나의 웹사이트를 만들수도 있다.
그래서 코드의 가독성을 고려해서 작업해야 한다.
나혼자만 보는 파일이 아니므로 다른 설명이 없어도 상호간에 구조를 알 수 있도록
그룹화가 잘되어 있고 의미 있는 네이밍으로 작성되어 있어야한다.

html은 하나의 문서이다. 문서는 보는 사람이 정확한 정보를 알 수 있어야 하며
지루하지 않은 구성으로 끝까지 읽을 수 있어야 한다.

html을 작성한다는 것은 결코 쉬운일도 가벼운 일도 아니다.
css를 활용하여 디자인된 화면을 보여주고 구조화된 내용을 사용자와 기계에 논리적으로 제공해야 한다.
또한 사용자가 어떠한 동작을 입력했을 때는 자바 스크립트를 활용하여 입력된 동작의 결과가
웹 브라우저에 나타나 사용자와의 상호작용이 이루어져야한다.
그래야 내가 만든 사이트가 더 오래 유지되고 어떤 브라우저와 기기에서도 모든 사용자가 웹사이트를 사용할 수 있다.
단순히 마크업만 익히는 웹 퍼블리셔보다는 사용자에게 서비스를 제공하는 웹 퍼블리셔가 되도록 노력하자

웹 퍼블리셔는 하나의 콘텐츠를 제작하는 콘텐츠 제작자이다.
사용자와 웹 브라우저에 보여지는 내용의 의미를 파악하여 의미에 맞는 태그를 사용하고 사용자에게 편한
UI/UX를 제공할 수 있도록 여러 가지를 고려해야 한다.
또한 내가 만든 콘텐츠가 어떤 역할을 하는지 어떤 결과를 가져와야 하는지도 고민하며 끊임없이 노력해야 한다.


html 기본 구조

<!DOCTYPE html> : HTML이 지켜야 하는 표준을 알려주는 DTD(Document Type Definition). 
모든 html 문서들은 dtd가 표기되어 있어야 하며
이것은 현재 사용하고 있는 html에 사용할 수 있는 요소나 속성을 정의 하는 기준.

<html> : html의 시작을 알려주는 태그, 웹문서 전체를 감싸고 있다.

<head> : <head> 태그에는 사이트의 제목인 <title> 태그를 비롯하여 문서의 정보를 담는 <meta> 태그
그리고 css 파일과 자바스크립트 파일이 있다.

<title>이지스퍼블리싱</title> : 현재 문서의 제목이며 웹 브라우저의 툴 바에 표시된다.
즐겨 찾기로 등록시 즐겨 찾기의 제목이 되며 검색엔진의 결과 제목으로 표시된다.

<meta charset="UTF-8"> : <meta> 태그는 브라우저에는 표현되지 않지만 현재 html의 모든 정보를 요약해서
담고있는 데이터. <title> 태그와 설명<description> 메타 태그는 검색 엔진이 페이지의 내용을 파악할 때 기본으로
사용하는 데이터이므로 페이지의 내용에 맞게 작성되어야 한다.

<script type="text/javascript" src="./js/jquery-3.2.1.min.js"></script> : 외부 자바스크립트 파일
src 속성으로 외부 파일을 가져와서 사용하는 방식(파일에 직접 작성 하는 경우에는 <script>태그안에 스크립트 작성)
 
<link rel="stylesheet" type="text/css" href="./css/ui.css"> : 외부 스타일 시트 파일
<link>태그를 사용하여 별도의 css파일을 연결하는 방식.

<body> : 브라우저에 표현되는 내용을 담고 있는 html의 본문 태그. DTD에 선언된 기준에 따라 유효성에 맞게
태그가 작성되어야 한다.

</html> : 본문 태그와 전체적이 html 작성을 마치고 사용하는 태그
